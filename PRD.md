# PRD 작성 실습 워크시트 기반 — 상세 PRD

> PRD v1.1 (2026-07-20) — 기존 v1.0의 장 구성과 문제·시장 배경·페르소나를 유지하면서, 실제 분석 노트북과 구현 가능한 MVP 범위에 맞춰 표현과 우선순위를 보완한 버전입니다.  
> 자료 조사가 필요한 수치는 실제로 검색·확인된 값만 기재했고, 확인하지 못한 항목은 "검증 필요"로 표기했습니다(추측·추정치로 채우지 않음).
> 분석 대상의 기본 명칭은 `텔레그램 주식정보 채널`로 통일하며, `리딩방`은 외부 통계·논문·기사의 원래 명칭이나 접근 제외 범위를 설명할 때만 사용합니다.

---

### 제품/기능명

```
텔레그램 주식정보 채널 언급종목 교차검증 MVP — "건전 투자 유도형" 과거 패턴 확인 도구
```

### 1. TL;DR — 가치 정의

| 구성 요소 | 내용 |
| --- | --- |
| **누가** | 공개 텔레그램 주식정보 채널에서 종목 정보나 추천성 메시지를 접하는 일반 개인투자자, 특히 급등주·소형주 관련 언급을 접했지만 가격·거래량·공시·뉴스를 직접 교차검증하기 어려운 투자자 |
| **왜** (해결할 문제) | 추천 메시지와 함께 뉴스가 제시되더라도, 일반 투자자는 언급 주변의 거래량이 평소보다 크게 증가했는지, 언급 후 가격이 시장보다 크게 하락했는지, 같은 기간 실제 공시가 있었는지, 유사한 기사가 반복됐는지를 한눈에 확인하기 어렵다. |
| **어떻게** (제품 사용 방법) | 기존에 수집된 공개 텔레그램 주식정보 채널 메시지에서 종목을 추출한 뒤, **코스닥 보통주 중 시가총액 300억 원 미만**을 초기 분석 대상으로 좁힌다. 이후 언급 주변 거래량 급증, 언급 전후 가격 흐름, 시장 대비 움직임을 핵심 신호로 분석하고 Open DART 공시와 네이버 뉴스는 보조 정황으로 제공한다. |

**Background 서술형**

```
개인 투자자 저변이 확대되는 가운데 불법 투자자문과 불법 투자 리딩방 피해도 지속적으로 발생하고 있다. 기존 종목 스크리너와 공시
서비스는 존재하지만, 텔레그램 주식정보 채널에서 언급된 특정 종목을 사건 단위로 묶어 언급·거래량·가격·시장·공시·뉴스의 시간적 패턴을
한눈에 비교하기는 어렵다. 동시에 FinanceDataReader·Open DART OpenAPI 등 공개 데이터가 제공되고 있어, 기존에 수집된
공개 텔레그램 주식정보 채널 메시지를 대상으로 과거 패턴 비교형 MVP를 구현할 수 있다. 본 MVP는 실시간 사전 탐지나 사기 판정을 약속하지 않고,
과거 사건에서 관찰된 패턴과 확인할 기준을 사용자에게 설명하는 데 초점을 둔다.
```

#### 분석 채널 범위와 용어 선정 배경

초기 프로젝트에서는 펌프앤덤프 의심 사례와 투자 리딩방을 직접 수집하는 방향을 검토했다. 그러나 VIP·유료 리딩방은 가입과 접근이 어렵고, 실제 펌프앤덤프 의심 채널은 메시지 삭제·비공개 전환·채널 폭파 등으로 데이터가 사라지는 경우가 많아 학생 프로젝트 수준에서 지속적이고 재현 가능한 표본을 확보하기 어려웠다.

따라서 MVP는 **리딩방이나 불법 채널이라고 확인되지 않은 공개 텔레그램 주식정보 채널**에서 수집된 메시지를 대상으로 한다. 이 가운데 종목 언급 주변에서 나타나는 거래량·가격·시장 대비 움직임을 분석하여 이상 패턴 후보를 탐색한다. 채널 자체를 리딩방·작전방·불법 채널로 분류하지 않으며, 개별 메시지와 종목 역시 시세조종 또는 펌프앤덤프 사례로 확정하지 않는다.

문서의 기본 용어는 `텔레그램 주식정보 채널`, `종목 언급`, `추천성 메시지`, `주의 패턴`으로 통일한다. 다만 외부 통계·논문·기사에서 공식적으로 `불법 투자 리딩방`이라는 용어를 사용한 경우에는 출처의 원래 명칭을 유지한다.

### 3. Problem Statement — 문제 정의

| # | 왜? | 답변 |
| --- | --- | --- |
| 1 | 사용자는 왜 이 상황에서 불편/불안을 느끼는가? | 텔레그램 주식정보 채널에서 종목 정보나 추천성 메시지를 접해도, 당시 거래량과 가격이 평소와 어떻게 달랐는지, 시장 전체 움직임을 제외해도 특이한 흐름이었는지 직접 확인하기 어렵다. |
| 2 | 왜 기존 대안으로는 해결이 안 되는가? | 기존 서비스에서는 시세·공시·뉴스가 각각 분리되어 있고, 채널 내 종목 언급일을 기준으로 전후 데이터를 사건 단위로 비교하려면 사용자가 직접 여러 서비스를 오가며 확인해야 한다. |
| 3 | 왜 이 문제가 단순 정보 제공이 아닌 제품(파이프라인) 기회인가? | 비정형 텔레그램 메시지에서 종목을 추출하고, 언급일을 기준으로 가격·거래량·지수·공시·뉴스를 같은 관찰 창에 정렬하는 작업은 수작업으로 반복하기 어렵다. 여러 데이터 소스를 자동으로 연결하는 분석 파이프라인이 필요하다. |
| 4 | 왜 이 문제가 지금 해결되어야 하는가? | 개인 투자자 참여가 확대되는 가운데 텔레그램 주식정보 채널에서 종목 정보가 빠르게 확산되고 있어, 사용자가 해당 정보를 그대로 따르기 전에 스스로 확인할 수 있는 기준을 제공할 필요가 있다. |
| 5 | 왜 우리가 이 문제를 해결하기 적합한가? | Open DART API, FinanceDataReader 등 오픈소스 라이브러리, 네이버 뉴스 검색 API의 무료 기본 제공 범위를 활용하면 별도의 유료 데이터 계약 없이 Colab 기반 실데이터 MVP를 구현할 수 있다. 설명 가능한 규칙을 핵심 분석 방식으로 사용하고 통계적 이상치 탐지는 보조 신호로 활용하여, 사용자가 각 상태의 근거 데이터와 계산 과정을 직접 추적·검토할 수 있도록 한다. |

| **Pipeline** | 분석 파이프라인이 재현 가능한가? | 기존 메시지 수집본 → 종목 추출 → 시세·공시·뉴스 → 결과 저장 완주율 | 오류 없이 전 단계 통과 | 매 실행 | Colab 실행 로그 |
| **Coverage** | 판단에 필요한 데이터가 충분한가? | 대상 사건 중 가격·거래량·지수 데이터가 모두 확보된 비율 | 데이터 확인 후 설정 | 매 실행 | 결과 테이블 결측치 집계 |
| **Explainability** | 결과 근거를 추적할 수 있는가? | 상태값에 수치·기준일·원문 링크가 함께 제공되는 비율 | 100% 지향 | 매 실행 | 결과 스키마 검증 |
| **Guardrail** | 과도한 단정이 발생하지 않는가? | 우량·부실·사기·매수·매도 확정 표현 노출 건수 | 0건 | 매 실행 | 문구 점검 |
| **Optional Backtest** | 주의 패턴이 과거 결과와 어떤 관련이 있는가? | 주의 패턴과 이후 시장 대비 하락의 연관성 | 검증 필요 | 표본 확보 후 | 시점 분리 백테스트 |

※ 실제 불법·사기 사건 정답 라벨이 충분하지 않으므로 `사기 탐지 정확도`, `부실기업 예측 정확도`, `급등 전 경고율`을 MVP의 확정 성과로 사용하지 않는다.

### 10. Constraints & Assumptions

#### 반드시 명시할 네 가지 해석 제한

1. **시가총액 300억 원 미만은 위험 기준이 아니다.** 코스닥 초소형주로 분석 대상을 줄이기 위한 초기 운영 기준이며, 확보 데이터의 범위·분포와 검증 결과에 따라 변경할 수 있다.
2. **거래량 배율과 가격 하락 임계값은 위험의 정의가 아니다.** 현재 데이터에서 후보를 나누기 위한 실험값이며 데이터 분포와 백테스트 결과에 따라 조정해야 한다.
3. **공시 미확인은 호재 부재·허위 기사·부실기업을 의미하지 않는다.** 관찰 기간 밖의 공시, 자율공시가 아닌 정보, 검색·매핑 누락 등이 있을 수 있다.
4. **본 화면은 불법·사기·정상 여부나 매수·매도 적정성을 판정하지 않는다.** 관찰된 과거 데이터와 사용자가 추가로 확인할 기준만 제공한다.

**제약 (바꿀 수 없는 한계)**

| 영역 | 제약 내용 |
| --- | --- |
| 분석 대상 | 코스닥 보통주 중 시가총액 300억 원 미만을 초기 기준으로 사용한다. 확보 데이터와 검증 결과에 따라 변경할 수 있으며, SPAC 등 구조가 다른 종목의 제외 여부는 추가 확인한다. |
| 시가총액 시점 | 현재 시점의 시가총액으로 과거 사건을 선정하면 사건 당시 규모와 달라질 수 있다. MVP에서는 사용한 시가총액 기준일을 반드시 표시하고, 가능하면 사건 당시 시가총액으로 보완한다. |

| DART | 현재 구현은 법인코드와 공시 목록을 조회하는 수준이다. 공시 존재 여부만으로 뉴스의 진위나 호재의 정상성을 판정할 수 없다. |
| 뉴스 | 제목·요약 기반 유사도와 표현 탐지는 기사 작성 경위나 매체 신뢰도를 증명하지 못한다. |
| 시장 데이터 | 거래정지, 신규 상장, 액면분할, 유상증자, 수정주가 처리 등에 따라 가격·거래량 비교가 왜곡될 수 있다. |
| 자원 | Google Colab 무료 티어를 전제로 하므로 대규모 상시 크롤링·서버 운영은 범위 밖이다. |
| 법무·윤리 | 결과는 참고용 과거 패턴이며 특정 기업·언론사·채널을 불법·사기·부실로 단정하지 않는다. 실제 종목명을 표시할 때는 기준일과 객관적 수치·원문을 함께 제공한다. |

**가정 (검증해야 할 전제)**

| 가정 | 검증 방법 |
| --- | --- |
| 코스닥 시가총액 300억 원 미만 종목에서 텔레그램 주식정보 채널 언급 주변 거래량 변화가 유의미한 분석 대상을 제공한다 | 대상 종목 수와 거래량 배율 분포를 확인하고 기준값 조정 |
| 거래량 급증과 언급 후 시장 대비 하락이 함께 나타나는 사건이 사용자에게 유용한 주의 기준이 된다 | 과거 사건 그룹 비교와 사용자 과업 테스트 |
| 공시와 뉴스 정황이 핵심 패턴의 맥락을 이해하는 데 도움을 준다 | 공시·뉴스를 포함/제외한 화면의 사용자 이해도 비교 |
| 사용자가 근거를 보고 유사 추천을 추가 확인하거나 추종매수를 재고한다 | 소규모 사용자 파일럿 설문·인터뷰 |

**선택적 재무 분석 범위**

재무정보 연계가 꼭 필요한 경우에도 메인 MVP와 분리된 P2 실험으로만 수행한다.

- 전체 대상 종목에 일괄 적용하지 않는다.
- 업종별 동일 기준으로 우량·부실을 판정하지 않는다.
- 메인 상태값을 올리거나 내리는 변수로 사용하지 않는다.
- `우량기업`, `부실기업`, `건전성 양호`, `상장폐지 가능성`과 같은 결과를 출력하지 않는다.
- 필요한 경우 일부 사례 상세에서 객관적으로 확인된 감사의견·관리종목 관련 공시의 제목·기준일·원문을 `재무 추가확인 정보`로만 제공한다.
- 별도 정답 라벨과 전문가 검토 없이 부실 예측 성능을 주장하지 않는다.

## 참고 — 이번 조사에서 확인하지 못해 빈칸으로 남긴 항목

- SAM/SOM(텔레그램 주식정보 채널의 정확한 이용자 규모, MVP 파일럿 목표 사용자 수)

- Success Metrics 각 지표의 구체적 목표치(파일럿 데이터 미확보)
- 경쟁 서비스가 "리딩방 추천 종목 매수 전 교차검증"을 정말 제공하지 않는지에 대한 전수 조사(현재는 조사 범위 내 미확인 수준)
- 실제 불법·사기 사건에 대한 신뢰할 수 있는 정답 라벨과 충분한 반례 데이터
- 공개 채널 메시지 및 실제 종목명 표시에 관한 법무 검토

위 항목은 추측치로 채우지 않고 비워 두었으며, 실데이터 분포 확인과 사용자 파일럿을 거쳐 갱신해야 합니다.
